## GLOBAL MOBILE DESIGN NOTES

> Apply these rules across all 8 screens:
>
> - **Status Bar**: Always 44px. White text/icons on Dark Navy screens, Dark Navy text/icons on white screens.
>
> - **Bottom Safe Area**: Always 34px padding below the last interactive element and below the bottom nav bar.
>
> - **Bottom Navigation**: Consistent across all Officer screens (Screens 6, 7, 8 and any subsequent officer screens). Never appears on auth screens (Screens 1, 2, 3).
>
> - **Back Navigation**: Auth screens (2, 3) always have a back arrow top-left returning to the Welcome Screen. Dashboard screens use the bottom nav — no back arrow needed.
>
> - **Tap Targets**: Every button, nav item, and interactive row minimum 48px height. No exceptions.
>
> - **Loading States**: Every login button shows a spinner and loading text for 1.5 seconds before navigating — makes the prototype feel realistic even without actual authentication.
>
> - **Floating Action Button**: Appears only on list screens (Events Screen 7). Always Violet, always 56px, always positioned 20px right and 80px bottom (above nav bar).
>
> - **Card Shadows**: Subtle on all raised white cards: `0 2px 8px rgba(0,0,0,0.06)`. More prominent on the floating action button: `0 4px 12px rgba(131,53,142,0.4)`.
>
> - **Typography Scale**: Page titles 22–26px bold Dark Navy. Card headers 15–16px bold Dark Navy. Body text 13–14px Dark Navy. Secondary text 12–13px Subtext Gray `#9E9E9E`. Labels and badges 11–12px.
>
> - **Color Usage Rule**: Golden Yellow `#FFD41C` is reserved for the Welcome Screen branding and the SAS Admin portal only — it signals institutional authority. Violet `#83358E` is the Officer portal color. Dark Navy `#001A4D` is the universal base. Never use Golden Yellow as a background fill on officer screens.
>
> - **Empty States**: Every list screen (Events, Finance, Members) has an empty state: centered Cobalt Blue Tabler icon (48px) + Dark Navy bold message + gray description + a Violet action button ("Create your first event" etc.).
>
> - **Prototype Fidelity**: All 8 screens must be connected per the Navigation Map at the top of this prompt. Every tap target that leads to another screen must have a prototype connection. The flow must be completable without dead ends.
>
> - Optimized for 375px × 812px (iPhone 14). All content works at 390px × 844px and 360px × 800px without layout breaking.
